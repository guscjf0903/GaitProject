## TS-005. Gradle kapt 컴파일 오류 (kaptGenerateStubsKotlin)

### 현상
- 백엔드 `./gradlew bootRun` 실행 시 `Internal compiler error`와 함께 `kaptGenerateStubsKotlin` 태스크에서 빌드 실패.

### 원인 분석
- Kotlin 1.9.20 환경에서 실제로는 사용하지 않는 `Lombok` 관련 플러그인과 `QueryDSL` 설정이 `build.gradle.kts`에 남아있어 Kotlin 컴파일러와 충돌 발생.

### 수정 내용
- **변경 파일**: `build.gradle.kts`
- **After**:
  - `lombok` 의존성 및 어노테이션 프로세서 완전 제거.
  - 코드 내에서 `JPAQueryFactory`를 사용하지 않으므로 불필요한 `querydsl-apt` 및 `kapt` 플러그인 설정 완전 제거. `QueryDslConfig.kt` 삭제.

## TS-006. API 재생성 후 프론트엔드 화면 백지화

### 현상
- `npm run generate:api` 명령어로 OpenAPI 클라이언트를 최신화한 직후, 브라우저 새로고침 시 워크스페이스 목록과 대화 내용이 전혀 렌더링되지 않음.

### 원인 분석
- 백엔드 컨트롤러 명세가 업데이트되면서, 프론트엔드에 생성된 서비스 클래스의 메서드명(Operation ID)이 미세하게 변경됨 (`CommitsService.list4` -> `list`, `UsersService.list` -> `list1`). 이로 인해 존재하지 않는 함수를 호출하여 JS 에러 발생.

### 수정 내용
- **변경 파일**: `src/pages/ChatPage.vue`, `src/pages/WorkspacePage.vue`
- **After**:
  - 템플릿 코드 내 구형 함수명을 새로 생성된 함수명으로 일괄 교체 (`list4` -> `list`).

## TS-007. DB에 토큰 사용량(total_tokens)이 계속 NULL로 들어감

### 현상
- `Message` 엔티티에 토큰 컬럼을 추가하고 저장을 시도했으나, DB 조회 결과 항상 `NULL`로 기록됨. 백엔드 로그에는 `EmptyUsage` 객체 반환.

### 원인 분석
- OpenAI API는 스트리밍(SSE) 요청 시 별도의 옵션을 주지 않으면 마지막 청크에 토큰 사용량(Usage) 메타데이터를 반환하지 않음. 
- Spring AI(1.0.3)의 `ChatResponse`는 데이터가 없으면 `EmptyUsage`를 반환하도록 설계되어 있었음.

### 수정 내용
- **변경 파일**: `config/.env.local`, `src/main/kotlin/com/gait/gaitproject/service/ai/OpenAiChatAiService.kt`
- **After**:
  - 환경변수 `SPRING_AI_OPENAI_CHAT_OPTIONS_STREAM_USAGE=true` 추가.
  - (확실한 제어를 위해) 코드 단에서 `OpenAiChatOptions.builder().model(defaultModel).streamUsage(true).build()`를 `Prompt`에 주입하여 명시적으로 API 스펙 요청.

## TS-008. 커밋 요약(Long Summary)이 'token' 등 단일 단어로 들어감

### 현상
- 사용자가 "token"이라고 커밋 제목을 입력하고 수동 커밋을 생성했을 때, 이전 대화를 기반으로 AI가 길게 요약해 줘야 할 `longSummary`에도 똑같이 "token"만 들어가고 AI 요약이 증발함.

### 원인 분석
- 백엔드 로그 확인 결과: `Commit summary AI failed. Fallback to empty summary. reason=HTTP 400 - invalid model ID`.
- 원인: TS-007 해결을 위해 `.env.local`에 환경 변수를 추가할 때, 개행(줄바꿈) 문자가 누락되어 기존 `APP_AI_COMMIT_SUMMARY_MODEL=gpt-4o-mini` 속성과 문자열이 하나로 합쳐짐 (`gpt-4o-miniSPRING_...`). 존재하지 않는 모델명으로 API를 호출하여 요약에 실패.
- AI 요약이 빈 값(`NULL`)을 반환하자, Fallback 로직에 의해 사용자가 입력한 짧은 제목(keyPoint)이 그대로 롱 서머리로 대체되어 저장됨.

### 수정 내용
- **변경 파일**: `config/.env.local`
- **After**:
  - 개행 문자를 삽입하여 두 환경변수를 올바르게 분리.
  - (추가 조치) 포트를 좀비처럼 점유하고 있던 이전 데몬 프로세스를 `lsof -t -i:8080 | xargs kill -9` 로 완벽히 사살한 후 서버를 깨끗하게 재시작하여 캐시된 설정을 날림.

## TS-009. graphPaths에 merge 엣지 추가 시 타입 에러 (isMergeEdge 불일치)

### 현상
- 머지 기능 구현 중 `ChatPage.vue`의 `graphPaths` computed에서 merge parent 엣지를 그리기 위해 `paths.push({ d, color, isActive, isMergeEdge: true })` 등을 추가했을 때, IDE/린터에서 `Object literal may only specify known properties, and 'isMergeEdge' does not exist in type '{ d: string; color: string; isActive: boolean }'` 에러 발생.

### 원인 분석
- `graphPaths` 내부에서 `paths` 배열을 `Array<{ d: string; color: string; isActive: boolean }>`로 선언해 두었기 때문에, 새로 추가한 `isMergeEdge` 필드가 타입에 없어 할당 불가로 처리됨.

### 수정 내용
- **변경 파일**: `GaitProject_frontend/src/pages/ChatPage.vue`
- **Before**:
  - `const paths: Array<{ d: string; color: string; isActive: boolean }> = []`
  - 일반 parent 엣지: `paths.push({ d, color, isActive })`
- **After**:
  - `const paths: Array<{ d: string; color: string; isActive: boolean; isMergeEdge: boolean }> = []`
  - 일반 엣지: `paths.push(..., isMergeEdge: false)`
  - merge 엣지: `paths.push(..., isMergeEdge: true)`
- 템플릿의 `<path>`에서 `:stroke-dasharray="path.isMergeEdge ? '6 4' : 'none'"` 등으로 사용.

### 검증
- `npx vue-tsc --noEmit` 통과. 머지 수행 후 그래프에서 parent 실선과 merge 점선이 구분되어 표시되는지 확인.
