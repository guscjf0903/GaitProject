### 2026-02-15 ~ 2026-02-16

#### AI 컨텍스트 및 브랜치/커밋 동작 개선

- **contextCommitId 도입**
  - `ChatStreamRequest`에 `contextCommitId` 추가
  - 체크아웃/타임트래블 시점의 커밋 기준으로 AI 컨텍스트 구성
  - `ContextBuilder`에서 `contextCommitId`가 있으면 Recent 영역 비움(미래 메시지 제외)

- **baseCommitId 도입**
  - `BranchCreateRequest`에 `baseCommitId` 추가
  - 과거 커밋에서 브랜치 생성 시 시작점 명시

- **브랜치 생성 로직**
  - `BranchService.create`: `baseCommitId`가 있으면 "Start {branchName}" 커밋 생성, 분기선 시각화
  - 프론트: `BranchesService.create2` 호출, `router.push`로 새 브랜치 URL 이동

- **커밋 목록 API**
  - `GET /api/workspaces/{wid}/branches/{bid}/commits` 추가
  - `CommitService.list`, `CommitController` 노출

- **타임라인 at-commit API**
  - `GET /api/workspaces/{wid}/branches/{bid}/messages/timeline/at-commit` 추가
  - `MessageService.timelineUpToCommit`: 조상 커밋 ID 수집 후 `findByWorkspace_IdAndCommit_IdIn` 호출

#### 트러블슈팅 (상세는 `docs/troubleshooting/2026-02.md` 참조)

- **timeline/at-commit 500**: `ids.isEmpty()` 시 `emptyList()` 반환
- **커밋 2~3번 생성**: `BranchRepository.findByIdForUpdate` (PESSIMISTIC_WRITE), pending 없으면 예외
- **초기 커밋 2개**: `WorkspaceService`에서 `initProject` 1개만 생성, 프론트 더미 커밋 제거
- **분기선 안 보임**: `BranchService`에서 "Start {name}" 커밋 생성

#### 프론트엔드 개선

- **watch(workspaceId, branchId)**: 라우트 변경 시 `bootstrapFromServer` 재실행, 새 브랜치 즉시 표시
- **loadingTimeline, committing**: 커밋 클릭 로딩 표시, 수동 커밋 중복 방지
- **timelineSeq, branchId, pendingCheckoutCommitId**: race 방지, 다른 브랜치 checkout 안정화
- **saveCurrentHeadSnapshot**: checkout 전 현재 화면 보존
- **loadMessagesLatest**: `Promise.all`로 커밋+미커밋 동시 요청, 첫 클릭에 최신 대화 표시
- **serverHeadCommitId**: 서버 기준 HEAD 판별, `isDetached` 정확도 향상
- **메시지 저장 실패 토스트**: USER/ASSISTANT 저장 실패 시 사용자 알림
- **로컬 스냅샷 유지**: 서버 응답이 로컬보다 짧으면 스냅샷 유지 + 토스트

#### DB 초기화

- `./scripts/db.sh reset local`: PostgreSQL 볼륨 삭제 후 재생성, 데이터 전체 초기화

### 2026-02 (최근 세션: 브랜치/커밋 UX 및 Git Graph 리팩터링)

#### 커밋 탐색 UX/로직 개선

- **하드 리셋형 커밋 탐색 제거**
  - 커밋 클릭 시 채팅 전체를 잘라서 재렌더링하지 않고, 현재 브랜치 선형 히스토리를 유지한 상태에서 해당 커밋 위치로 포커스 이동하도록 변경
- **커밋 포커스 UX 강화**
  - `focusCommitInChat()` 추가로 부드러운 스크롤 + 임시 하이라이트 적용
- **미래 메시지 시각화**
  - 과거 커밋 선택 시 이후 메시지/커밋 구간을 반투명 처리
- **브랜치 분기점 힌트**
  - 커밋 divider에 `🔀 +N branches` 배지/툴팁/브랜치 이동 액션 추가

#### 브랜치 컨텍스트 일관성 강화

- **공통 조상 클릭 시 브랜치 유지**
  - B 브랜치에서 A2(공통 조상) 클릭 시 A로 강제 이동하던 문제 수정
  - 현재 브랜치 HEAD 조상 여부로 판단하여, 공통 조상은 현재 브랜치 컨텍스트 유지
- **최신 커밋 첫 클릭 시 최신+미커밋 즉시 표시**
  - 첫 클릭에 커밋분만 보이고 두 번째 클릭에서 미커밋이 붙는 문제를 로직적으로 제거

#### 커밋 표시 위치 일관화

- **커밋 직후 vs 새로고침 후 표시 위치 불일치 수정**
  - 기존: 커밋 직후에는 divider가 끝 지점에 붙고, 새로고침 후에는 시작 지점에 표시
  - 개선: 커밋 직후에도 서버 기준으로 타임라인 재구성하여 항상 “커밋 시작 지점”에 표시

#### Git Graph 렌더링 리팩터링

- **트랙(컬럼) 할당 알고리즘 도입**
  - 브랜치별 고유 트랙을 할당(`branchTrackMap`, `commitTrackMap`)
  - x좌표를 커밋 배열 임시 값이 아닌 트랙 기반으로 계산
- **트랙 간 간격 확대**
  - `TRACK_GAP` 상수 도입 및 적용
- **우회 경로 계산**
  - 분기/합류 시 중간 빈 트랙을 우선 사용해 선 겹침을 줄이는 `pickViaTrack()` 라우팅 적용
- **레이어/강조 개선**
  - 선(path)을 배경 레이어로 유지, 노드는 전경 강조
  - 현재 선택 라인/노드 강조, 비활성 라인/노드 감쇠 처리

#### 사이드바/그래프 레이아웃 동적화

- **사이드바 접기/펼치기 + 폭 리사이즈**
- **그래프 패널 별도 리사이즈**
  - 그래프 영역과 커밋 리스트 영역을 독립적으로 조절 가능하게 변경
- **자동 최소폭 보정**
  - 트랙 수가 늘어날 때 그래프 패널 최소폭을 자동 보정해 잘림 완화
- **상태 저장**
  - 사이드바 폭/접힘/그래프 패널 폭을 `localStorage`에 저장/복원

#### 품질 확인

- 프론트 `npm run build` 반복 검증 통과
- 변경 파일 lint 에러 없음 확인
